# web

Admin web interface for the access control system.

## Overview

A single-page application (SPA) for managing readers, doors, cards, and viewing audit logs. Communicates with the backend via REST API.

## Features

### 1. Reader Management
- View all registered readers with key versions
- Add/update reader with key version
- Bind doors to readers
- Delete readers

### 2. Door Role Management
- View door-to-role mappings
- Allow a role to access a door
- Revoke role access from a door

### 3. Card Management
- View all registered cards (shows HMAC, not raw ID)
- Add card with role assignment
- Delete card by HMAC

### 4. Audit Log
- View access attempt history
- Verify cryptographic chain integrity
- Shows: timestamp, reader, door, allow/deny, reason

### 5. Runtime Logs
- Real-time event polling (every 500ms)
- Formatted log output with timestamps
- Shows frame, decision, audit, and admin events

### 6. Database Management
- Export full database as SQLite file
- Import database from file (replaces current)

### 7. Scan Simulation
- Test access decisions without hardware
- Specify card ID, reader ID, door ID
- Returns allow/deny result with reason

## Authentication

The admin panel uses token-based authentication:

1. Enter token in the sidebar input
2. Click "Save token" (stores in localStorage)
3. All API requests include `X-Admin-Token` header

Configure the required token in `config/access_security.yaml`:

```yaml
admin:
  adminToken: "your-secret-token"
```

## Serving Static Files

The web files are served by the C++ admin server:

```cpp
// Static file routes in access_admin
svr.set_mount_point("/static", "web");
svr.set_mount_point("/", "web");
```

Access at: `http://localhost:8080/`

## Development

### Local Testing

1. Start the access_admin server:
   ```bash
   ./build/access_admin/access_admin config/access_security.yaml
   ```

2. Open browser to `http://localhost:8080/`

### Modifying Styles

Edit `css/styles.css` - changes apply on page reload.

### Adding New Functionality

1. Create new JS file in `js/` directory
2. Add script tag to `index.html` (after `utils.js`)
3. Add navigation button to sidebar
4. Create new `<section>` in main content
