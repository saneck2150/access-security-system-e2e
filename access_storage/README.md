# access_storage

SQLite-based persistent storage for access control data and tamper-evident audit logging.

## Components

| File | Purpose |
|------|---------|
| `sqlite_access_store.hpp/cpp` | Card, reader, door role CRUD operations |
| `sqlite_audit_log.hpp/cpp` | HMAC-chained audit event logging |
| `audit_verify.hpp/cpp` | Audit chain integrity verification |
| `serialization.hpp/cpp` | Shared little-endian encoding utilities |
| `sqlite_helpers.hpp` | SQLite RAII wrappers and helpers |

## SQLite Configuration

On open, the store sets:
- `PRAGMA journal_mode=WAL` — allows concurrent reads with one writer
- `PRAGMA busy_timeout=5000` — waits up to 5s instead of immediate SQLITE_BUSY
- `PRAGMA foreign_keys=ON` — enforces referential integrity

## Audit Chain Integrity

Each audit entry is chained via HMAC:

```
┌──────────────────────────────────────────────────────────────────┐
│                    Audit Chain Structure                          │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│   Entry 1                Entry 2                Entry N           │
│  ┌─────────┐           ┌─────────┐           ┌─────────┐         │
│  │prev_hash│──zeros    │prev_hash│───────────│prev_hash│         │
│  │   ...   │           │   ...   │           │   ...   │         │
│  │entry_hash├──────────►entry_hash├──────────►entry_hash│         │
│  └─────────┘           └─────────┘           └─────────┘         │
│       │                                            │              │
│       └─── HMAC(key, prev_hash || event_fields) ───┘              │
│                                                                   │
│                                              ┌─────────────┐      │
│                         must match ─────────►│audit_anchor │      │
│                                              │ last_hash   │      │
│                                              └─────────────┘      │
└──────────────────────────────────────────────────────────────────┘
```

**Hash computation:**
```
entry_hash = HMAC-SHA256(key, prev_hash || ts || reader_id || door_id || seq || allow || reason || card_hmac || action)
```

## Usage

### Access Store
```cpp
#include <access_storage/sqlite_access_store.hpp>

access_storage::SqliteAccessStore store("data/access.db");
store.initSchema();

// Card management
store.upsertCardHmac("abc123...", "employee");
auto role = store.roleForCardHmac("abc123...");

// Reader management  
store.upsertReader(1, 1);  // reader_id=1, key_version=1
store.allowDoorForReader(1, 100);  // reader 1 can control door 100

// Door roles
store.allowRole(100, "employee");
bool allowed = store.isAllowed(100, "employee");
```

### Audit Log
```cpp
#include <access_storage/sqlite_audit_log.hpp>

std::array<uint8_t, 32> hmacKey = /* ... */;
access_storage::SqliteAuditLog audit(store.dbHandle(), hmacKey);

access_decision::AuditEvent event{
    .ts_unix_ms = 1234567890000,
    .reader_id = 1,
    .door_id = 100,
    .seq = 42,
    .allow = true,
    .reason = "ok",
    .card_id = "abc123...",
    .action = "open"
};
audit.append(event);
```

### Verification
```cpp
#include <access_storage/audit_verify.hpp>

auto result = access_storage::verifyAuditChain(db, hmacKey);
if (!result.ok) {
    std::cerr << "Chain broken at row " << result.bad_id 
              << ": " << result.error << "\n";
}
```
