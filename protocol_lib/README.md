# protocol_lib

Binary protocol library for secure access control frame encoding, decoding, and replay protection.

## Overview

This module handles the wire format for communication between card readers and the access control server. It provides serialization/deserialization of encrypted frames and replay attack detection.

## Wire Format

All multi-byte integers are **little-endian**.

```
┌────────────────────────────────────────────────────────────────────┐
│                         Frame Layout                                │
├──────────┬──────┬───────────────────────────────────────────────────┤
│  Offset  │ Size │ Field                                             │
├──────────┼──────┼───────────────────────────────────────────────────┤
│     0    │   4  │ Magic ("AS01")                                    │
│     4    │   1  │ Version (0x01)                                    │
├──────────┼──────┼───────────────────────────────────────────────────┤
│     5    │   4  │ reader_id (uint32)                                │
│     9    │   4  │ door_id (uint32)                                  │
│    13    │   8  │ ts_unix_ms (uint64)                               │
│    21    │   8  │ seq (uint64)                                      │
│    29    │   4  │ key_version (uint32)                              │
│    33    │  24  │ nonce (XChaCha20)                                 │
├──────────┼──────┼───────────────────────────────────────────────────┤
│    57    │   4  │ ciphertext_length (uint32)                        │
│    61    │   N  │ ciphertext (N bytes)                              │
│   61+N   │  16  │ Poly1305 tag                                      │
└──────────┴──────┴───────────────────────────────────────────────────┘

Minimum frame size: 77 bytes (with 0-byte ciphertext)
```

## Usage Example

### Parsing an incoming frame

```cpp
#include "protocol_lib/frame.hpp"
#include "protocol_lib/replay_window.hpp"

protocol::replay::ReplayWindow replay(256);

// Parse raw bytes
auto frame = protocol::frame::parseFrame(raw_bytes);

// Check for replay attack
if (!replay.accept(frame.header.seq)) {
    // Reject: replay detected
}

// Decrypt frame.ct using frame.header.nonce and frame.tag
// ...
```

### Serializing an outbound frame

```cpp
#include "protocol_lib/frame.hpp"

protocol::frame::Frame f;
f.header.reader_id = 1;
f.header.door_id = 100;
f.header.ts_unix_ms = now_ms();
f.header.seq = next_seq++;
f.header.key_version = 1;
// ... set nonce, ct, tag after encryption

auto wire_bytes = protocol::frame::serialize(f);
```

## Security Notes

- **Replay Protection**: `ReplayWindow` maintains a sliding window of seen sequence numbers. Configure window size based on expected message rate and acceptable latency.
- **DoS Protection**: `parseFrame()` accepts a `maxCtLen` parameter (default 4096) to prevent memory exhaustion from malformed frames.
- **Authenticated Encryption**: The 16-byte Poly1305 tag ensures integrity; verification happens in the crypto layer, not here.
