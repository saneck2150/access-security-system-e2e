## HKDF Key Derivation

Implements RFC 5869 HKDF using HMAC-SHA256:

```
                    ┌─────────────────────────────────────┐
                    │            HKDF-SHA256              │
                    ├─────────────────────────────────────┤
                    │                                     │
   IKM (master key) ─┬─►┌─────────┐                       │
                     │  │ Extract │──► PRK (32 bytes)     │
   Salt (optional) ──┘  └─────────┘        │              │
                                           ▼              │
                        ┌─────────────────────────────┐   │
   Info (context) ─────►│          Expand             │   │
                        │  T(1) = HMAC(PRK, info||1)  │   │
                        │  T(2) = HMAC(PRK, T(1)||info||2) │
                        │  ...                        │   │
                        └─────────────────────────────┘   │
                                           │              │
                                           ▼              │
                               OKM (derived key)          │
                    └─────────────────────────────────────┘
```

**Usage in system:**
- `key_manager` uses HKDF to derive purpose-specific keys from master key
- Info string provides domain separation ("aead-kv1", "card-pepper", etc.)

## Authenticated Encryption (AEAD)

XChaCha20-Poly1305 with automatic nonce management:

```
                   ┌───────────────────────────────────────┐
                   │            SecureAead                 │
                   ├───────────────────────────────────────┤
                   │                                       │
  Plaintext ──────►│  ┌─────────────────────────────────┐  │
                   │  │     XChaCha20-Poly1305          │  │
  AAD ────────────►│  │                                 │  │──► Ciphertext
                   │  │  nonce = prefix (16) || seq (8) │  │
  Key ────────────►│  │                                 │  │──► Tag (16 bytes)
                   │  └─────────────────────────────────┘  │
                   │                                       │──► Nonce (24 bytes)
                   │     seq++ after each seal()           │
                   └───────────────────────────────────────┘
```

**Properties:**
- 256-bit key (32 bytes)
- 192-bit nonce (24 bytes) = random prefix (16) + sequence (8)
- 128-bit authentication tag (16 bytes)
- AAD is authenticated but not encrypted

## Security Considerations

| Property | Implementation |
|----------|----------------|
| Key size | 256-bit (XChaCha20-Poly1305 requirement) |
| Nonce uniqueness | Random prefix + incrementing sequence counter |
| Nonce overflow | Throws `std::runtime_error` at UINT64_MAX |
| Auth failure | Throws `std::runtime_error` (no partial plaintext) |
| Side channels | Uses libsodium's constant-time primitives |
