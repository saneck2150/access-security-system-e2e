# data

Runtime data directory for SQLite databases and persistent storage.

## Overview

This directory contains the SQLite database files generated and used by the access control system at runtime. It is excluded from version control (via `.gitignore`) to prevent committing sensitive access control data.

## Contents

| File | Description |
|------|-------------|
| `access.db` | Main SQLite database (created automatically on first run) |

## Configuration

The database path is configured in `config/access_security.yaml`:

```yaml
storage:
  sqlitePath: "data/access.db"
```

Default: `data/access.db` (relative to working directory)

## Backup & Restore

### Export via Admin API

```bash
curl -H "X-Admin-Token: <token>" \
     http://localhost:8080/api/db/export \
     -o backup.db
```

### Import via Admin API

```bash
curl -X POST \
     -H "X-Admin-Token: <token>" \
     -F "db=@backup.db" \
     http://localhost:8080/api/db/import
```

### Manual Backup

```bash
# While server is running (uses SQLite backup API)
sqlite3 data/access.db ".backup 'backup.db'"

# Or simply copy when server is stopped
cp data/access.db backup.db
```

## Security Notes

- **Sensitive Data**: The database contains card HMACs and access logs. Protect with filesystem permissions.
- **Audit Integrity**: The `audit_log` table uses hash chaining. Use `audit_verify` tool to check for tampering.
- **Excluded from Git**: Listed in `.gitignore` to prevent accidental commits.
