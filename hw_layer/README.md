# hw_layer

ESP32-based RFID card reader firmware for the access control system.

## Overview

This firmware runs on an ESP32 microcontroller connected to an MFRC522 RFID reader. When a card is scanned, it sends the card UID to the access control server via HTTP and signals the result (allow/deny) through LED and buzzer feedback.

## Hardware Requirements

| Component | Description |
|-----------|-------------|
| ESP32 | Development board (e.g., ESP32-WROOM-32, ESP32-DevKitC) |
| MFRC522 | 13.56 MHz RFID reader module |
| LED | Green LED with 220Ω resistor |
| Buzzer | Active or passive piezo buzzer |
| Wires | Jumper wires for connections |

> ⚠️ **Important**: MFRC522 operates at 3.3V. Do NOT connect to 5V or you may damage the module.

## Software Setup

### Option 1: Arduino IDE

1. **Install Arduino IDE** (v2.x recommended)
   - Download from https://www.arduino.cc/en/software

2. **Add ESP32 Board Support**
   - Go to `File` → `Preferences`
   - Add to "Additional Board Manager URLs":
     ```
     https://espressif.github.io/arduino-esp32/package_esp32_index.json
     ```
   - Go to `Tools` → `Board` → `Boards Manager`
   - Search "esp32" and install "esp32 by Espressif Systems"

3. **Install Required Libraries**
   - Go to `Sketch` → `Include Library` → `Manage Libraries`
   - Search and install:
     - `MFRC522` by GithubCommunity

4. **Select Board**
   - Go to `Tools` → `Board` → `ESP32 Arduino` → `ESP32 Dev Module`

### Option 2: PlatformIO

1. **Install PlatformIO**
   - Install VS Code extension "PlatformIO IDE"

2. **Create platformio.ini** (if not exists):
   ```ini
   [env:esp32dev]
   platform = espressif32
   board = esp32dev
   framework = arduino
   lib_deps =
       miguelbalboa/MFRC522@^1.4.10
   monitor_speed = 115200
   ```

### Timing Behavior

| Event | Duration |
|-------|----------|
| WiFi connection timeout | 20 seconds |
| Card debounce (same card) | 1.5 seconds |
| Allow signal (LED + beep) | 180 ms |
| Deny signal (2x blink + beep) | ~560 ms |
| Main loop delay | 30 ms |

## LED & Buzzer Signals

| Signal | LED Pattern | Buzzer | Meaning |
|--------|-------------|--------|---------|
| Allow | Solid ON (180ms) | 2kHz beep | Access granted |
| Deny | 2x blink | 2x 500Hz beep | Access denied |
| No signal | OFF | Silent | Idle / WiFi connecting |

## Security Considerations

- WiFi credentials are stored in plaintext in firmware
- HTTP (not HTTPS) is used for server communication
- Admin token is transmitted in headers
- For production, consider:
  - Using HTTPS with certificate validation
  - Storing credentials in ESP32 NVS (encrypted)
  - Implementing encrypted payload (see `protocol_lib`)
